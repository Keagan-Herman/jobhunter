### Anti-Patterns Verdict

The PRODUCT.md names "Generic SaaS cream: warm-neutral backgrounds" as anti-reference #1. The entire app is built on #f8f8f4 — that exact warm-tinted near-white. The secondary tell is the uniform uppercase mono label system used across every element below heading level. CLI detector returned zero findings (clean pass on automated checks). No browser visualization attempted.

### Overall Impression

Solid architecture and a genuinely good score visualization system. But the visual language contradicts the design brief: the product wants to feel like Raycast, and it looks like a polished Notion template in SaaS cream with every label in uppercase mono.

### What's Working

1. Score visualization is on-brand and informative — RadialScore + EXCEPTIONAL/STRONG/COMPATIBLE/POTENTIAL labels deliver real signal at a glance.
2. Animation system is technically correct — stagger delays, strong easing curves, prefers-reduced-motion handled globally.
3. Semantic z-index scale and contrast ratios are documented in CSS custom properties.

### Priority Issues

**[P1] Cream background is the primary AI slop signal**
Background #f8f8f4 runs across every surface including fixed overlays. Explicitly listed as anti-reference in PRODUCT.md. Fix: dark near-neutral or true white #ffffff with gold as identity carrier.

**[P1] Uppercase mono is the only text register**
font-mono uppercase tracking-[2px/3px] used for metadata, labels, button text, tab names, eyebrows, and descriptions alike. Hierarchy collapses. Fix: reserve uppercase mono for status labels and badges; use mixed-case sans at 12-14px for names, descriptions, and metadata.

**[P1] Copy is bureaucratic for a personal daily-use tool**
"Initiate", "Recalibrate", "SEARCH ARCHIVE", "Configuration Complete", "No Pending Records". Fix: "Find new jobs", "Re-score", "Search your matches…", "Your setup is complete", "No matches yet".

**[P2] Detail panel is inline-below, not alongside**
Clicking a job card requires scrolling past the entire list to reach the panel. On mobile the selected card leaves the viewport entirely. Fix: grid side-by-side on desktop; bottom sheet on mobile.

**[P2] No keyboard navigation for a Raycast-precision product**
Arrow keys do nothing. No Escape to close panel. No shortcuts for Scan or Apply. Fix: arrow key card navigation, Enter to open, Escape to close, keyboard hint near scan button.

### Persona Red Flags

**Alex (Power User):** No keyboard navigation, no shortcuts, no score/filter controls, no undo for Apply.
**Casey (Mobile):** Detail panel pushes card off-screen; typed text uppercased by CSS; stat cards too padded on small screens; tab overflow gives no swipe affordance; hover-only clear button on search.

### Minor Observations

- bg-klimt-gold gradient utility is one clip-text usage away from an absolute ban violation — constrain or remove
- placeholder:text-[#888] on #f8f8f4 achieves ~3.3:1 contrast, below 4.5:1 threshold
- Search input uppercase class uppercases user-typed text visually, not just the placeholder
- text-glow-gold is decorative text-shadow with no state signal — cut if not marking a state
- Empty state emojis are aria-hidden correctly but identical text makes tab differentiation label-only
